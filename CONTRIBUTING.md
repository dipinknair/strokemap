# Contributing to Strokemap

Thank you for your interest in contributing to Strokemap! We welcome bug reports, documentation improvements, tests, and new features.

## How to Contribute

1. Fork the repository and create a branch from `main`.
2. Open an issue if you want to discuss a new feature or bug before working on it.
3. Create a pull request with a clear title and description.
4. Reference any related issue in the PR description.

## Code Style

- Use Python 3.10+ compatible syntax.
- Keep line lengths reasonable and readable.
- Follow the existing formatting and naming conventions.
- Run `ruff` and `pytest` before opening a PR.

## Testing

- Add regression tests for bug fixes and new features in `tests/`.
- Keep test cases focused and small.
- Run tests using:

```bash
pytest -v
```

## Review Process

- PRs are reviewed for correctness, style, and maintainability.
- Reviewers may request changes before merge.
- Be responsive to review comments and update your branch as needed.
