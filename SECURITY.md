# Security Policy

If you discover a security vulnerability in Strokemap, please do not create a public issue.

## Reporting a Vulnerability

1. Open a new private security issue if GitHub supports private vulnerability reporting.
2. If private reporting is unavailable, create a regular issue and clearly mark it as a security report.
3. Include as much detail as possible without sharing sensitive secrets.

## Response

The maintainers will review and respond as soon as possible. Please allow time for verification and coordination before public disclosure.

## Supported Versions

This repository is currently under active development. Security fixes are generally provided on the latest `main` branch.
